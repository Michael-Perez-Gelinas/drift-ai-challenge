# Owner UI Build Brief (Phases A‚ÄìE)

Shared contract for all agents building the Drift owner app. The data layer
(`src/lib/daily-log/repository.ts`, `stats.ts`) and schema already exist and are
tested. This brief defines the UI, routing, server-actions layer, and design
conventions. **No customer-facing work** ‚Äî owner app only, but server actions
must persist complete data (incl. geocoded lat/lng) so the customer site can
read it later.

## Design principles (non-negotiable)

- **Intentional minimalism.** No decorative elements. Every element earns its place.
- **Typography carries the design** ‚Äî weight/size/spacing for hierarchy, not chrome.
- **No generic card grids.** Don't default to a grid of rounded rectangles.
- **Mobile-first, 390px.** Large tap targets (min 44px), one-handed use, one primary
  action per screen. No hover-dependent interactions.

## Brand tokens (already in `tailwind.config.ts`)

- Fonts: `font-display` = Caveat (headings/display), `font-body` = DM Sans (everything else).
- Palette: cream/sand/clay/rust/dusk/sage/ink. Background `bg-background` (#F4EDE3),
  surfaces `bg-surface-card` / `bg-surface-raised`. Text `text-text-primary` (#1A1410),
  `text-text-secondary`, `text-text-muted`. Brand primary = rust (`text-brand-primary`,
  `bg-rust-500`). Accent = dusk. **No pure black** ‚Äî use `ink-900`.
- Admin nav: `bg-admin-nav-bg` (#1A1410), `text-admin-nav-text`, active = `admin-nav-active` (rust #C46B42).
- Status: `status-sold-out-*`, `status-open-*`, `status-closed-*`.
- Shadows: `shadow-card` / `shadow-md` (warm clay-tinted). Radius: `rounded-card` (12px), `rounded-lg`.
- Spacing scale is 4/8px-based (Tailwind `1`=4px ‚Ä¶ `6`=24px ‚Ä¶). Use the scale, no arbitrary values.
- Icons: `lucide-react`. Illustrations: hand-drawn SVGs come later ‚Äî leave nothing blocking on them.

## Routing & structure

```
src/app/login/page.tsx                 # password form (public)
src/app/api/auth/login/route.ts        # POST: check ADMIN_PASSWORD, set cookie
src/app/api/auth/logout/route.ts       # POST: clear cookie
src/app/admin/layout.tsx               # auth guard + bottom nav frame
src/app/admin/today/page.tsx           # Today (default landing)
src/app/admin/menu/page.tsx            # Menu management
src/app/admin/history/page.tsx         # History list
src/app/admin/history/[date]/page.tsx  # Day detail (editable)

src/app/admin/actions.ts               # 'use server' actions (the data exchange)
src/components/admin/                   # shared admin UI kit + per-screen components
```

- `/` should redirect to `/admin/today` (owner app is the product for now).
- Auth: middleware already guards `/admin/*` ‚Üí `/login`. Login sets an httpOnly cookie
  `drift_admin_session=authenticated` via `src/lib/auth/session.ts` (already exists:
  `setAdminSession`, `clearAdminSession`, `isAuthenticated`). `admin/layout.tsx` also
  calls `isAuthenticated()` and redirects to `/login` if false (defense in depth).
- Server Components by default; `'use client'` only for interactivity (toggles, forms, nav active state).

## "Today" date helper

Boston timezone. Provide `src/lib/daily-log/today.ts`:
```ts
export function todayISO(): string {
  // YYYY-MM-DD in America/New_York
  return new Intl.DateTimeFormat("en-CA", { timeZone: "America/New_York" }).format(new Date());
}
```

## Server-actions layer ‚Äî `src/app/admin/actions.ts`

All actions: `'use server'`, first line `await requireAuth()`, use `createServiceClient()`
from `src/lib/supabase/server.ts`, call the repository, then `revalidatePath(...)` the
affected route. `requireAuth` = `if (!(await isAuthenticated())) redirect('/login')`.

Repository functions available (in `src/lib/daily-log/repository.ts`):
`postLocation(db,{date,address,note?})`, `toggleSoldOut(db,{itemId,soldOut,today})`,
`wrapUpDay(db,{date,revenueCents?,customerCount?,endOfDayNote?,perItemUnits?})`,
`listDays(db)`, `getDay(db,date)`.

Actions to implement:
- `postLocationAction({ address, note })` ‚Äî geocode address server-side via Nominatim
  (`https://nominatim.openstreetmap.org/search?format=json&q=<addr>&limit=1`, send a
  `User-Agent: drift-app`); call `postLocation(db,{date: todayISO(), address, note})`;
  then `db.from('locations').update({ lat, lng }).eq('date', todayISO())` if geocode
  succeeded (fall back to no lat/lng on failure ‚Äî never block). revalidate `/admin/today`.
- `markClosedAction()` ‚Äî `db.from('locations').update({ is_open:false }).eq('date', todayISO())`. revalidate.
- `reopenAction()` ‚Äî set `is_open:true` for today. revalidate.
- `toggleSoldOutAction(itemId, soldOut)` ‚Äî `toggleSoldOut(db,{itemId,soldOut,today:todayISO()})`. revalidate `/admin/today`.
- `wrapUpDayAction(input)` ‚Äî `wrapUpDay(db, input)` with `date: todayISO()` (or a passed date for History edits). revalidate `/admin/today` and `/admin/history`.
- `updateDayAction(date, input)` ‚Äî same as wrapUpDay but for an arbitrary past date (History edit). revalidate `/admin/history/[date]`.
- Menu CRUD (direct table ops via service client):
  - `addMenuItemAction({ name, description, price, category, image_url })` ‚Äî price in **cents** (integer).
  - `updateMenuItemAction(id, fields)`.
  - `archiveMenuItemAction(id)` ‚Äî set `is_archived:true` (soft delete). revalidate `/admin/menu`.

Money: always integer cents. Display `(price/100).toFixed(2)` or use `formatRevenue`
from `stats.ts` for whole-dollar revenue.

## Shared admin UI kit ‚Äî `src/components/admin/`

Build these so the three screens compose a consistent app (foundation agent creates them):
- `BottomNav` (client) ‚Äî fixed bottom bar, 3 tabs Today | Menu | History with lucide icons
  (`MapPin`, `UtensilsCrossed`, `CalendarClock`), active tab in rust, dark nav bg. Uses `usePathname`.
- `ScreenHeader` ‚Äî Caveat title + optional subtitle + optional right action slot. Sets the type rhythm.
- `PrimaryButton` / `SecondaryButton` ‚Äî full-width-friendly, 44px+ tall, rust primary.
- `Field` ‚Äî labeled input/textarea wrapper (DM Sans label, warm border, focus ring rust).
- `Toggle` ‚Äî large switch for sold-out (44px target).
- `EmptyState` ‚Äî Caveat headline + muted line, for "not posted yet" / "no items".
- Keep components small and focused. No external UI lib beyond what's installed (`@base-ui/react`, lucide).

## Per-screen requirements

### Today (`/admin/today`) ‚Äî the heart, used at the truck
States driven by today's `locations` row (fetch via service client, `date = todayISO()`):
- **Not posted yet:** prominent `EmptyState` + a "Post today's location" form (address field + optional
  note, single submit ‚Üí `postLocationAction`).
- **Posted:** a summary (today's address + note) with edit + a "Mark closed"/"Reopen" control.
- **Sold-out toggles:** list every available menu item (`is_archived=false`), grouped by category,
  each with a large `Toggle` bound to `toggleSoldOutAction`. Reflect current `is_sold_out`.
- **"Wrap up today"** (collapsed by default; expandable): revenue ($ ‚Üí cents), customer count,
  optional per-item units (only for items you want to count), end-of-day note ‚Üí `wrapUpDayAction`.
  Prefill from today's `daily_performance` if already wrapped.

### Menu (`/admin/menu`) ‚Äî management, used at home
- Item list grouped by category (`is_archived=false`), showing name, price, category.
- Add new item (name, description, price, category, image_url) ‚Üí `addMenuItemAction`.
- Edit inline/sheet ‚Üí `updateMenuItemAction`. Archive (not hard delete) ‚Üí `archiveMenuItemAction`.

### History (`/admin/history`) ‚Äî showcases the logged data
- Reverse-chron list via `listDays(db)`: each row = date (friendly format), address, open/closed badge,
  and revenue via `formatRevenue` ‚Äî show "No numbers logged" when `revenue_cents` is null (never $0).
- Tap a row ‚Üí `/admin/history/[date]` detail via `getDay(db, date)`: location, revenue, customers,
  **top seller** (derive with `topSeller` from `stats.ts`), sold-out items, end-of-day note.
  All numbers editable via `updateDayAction`.
